## Bugs

- [x] (md-link-scan-quadratic) MarkdownRenderer inline link parsing was O(n²) on bracket-heavy input — `Preview/MarkdownRenderer.swift`: each unmatched `[` re-scanned to EOF for `]`/`)`. Fixed by precomputing per-`parseNodes`-invocation nearest-`]`/`)` index arrays (`nextCloseBracket`/`nextCloseParen`, built only when the slice contains `[`), turning `parseLink`'s two to-EOF scans into O(1) lookups → true O(n), covering all three pathological families incl. `[`×N + trailing `]`. Provably a no-op memoization: byte-identical output verified by a new seeded differential fuzz in `scripts/MarkdownRendererTests/main.swift` (diffs tree + render output + anchors vs a naive reference) plus the unchanged 133 assertions. Found in markdown-renderer adversarial review. (shipped 2026-07-20)
